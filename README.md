# 박준모 | Backend Developer, Aspiring Polyglot Full-stack Architect

> **"모르면 커피를 사주며 물어보고, 키워드를 적어놓고, 따로 찾아보고, 습득할 때까지 반복한다."**

방황하던 시절, 개발을 만나면서 처음으로 배움의 이유를 찾았습니다.

C++로 시스템 프로그래밍을 하던 시절부터, Java/Spring Boot로 백엔드를 구축하고, 지금은 Python과 AI 생태계까지 영역을 넓히고 있습니다. 하나의 언어에 갇히지 않는 폴리글랏 개발자, 코드만 짜는 것이 아니라 구조를 설계하는 풀스택 개발자가 목표입니다.

---

## 나에 대해

모르면 물어봅니다. 이해가 안 되면 "왜?"를 끝까지 파고듭니다. 하나의 개념을 이해하면 그 뒤에 숨은 원리가 궁금해지고, 그 원리를 이해하면 또 다른 "왜?"가 떠오릅니다. 이 연쇄적인 질문이 저를 성장시키는 엔진입니다.

이 사이클을 반복하면서 지금의 기술 스택을 쌓았습니다.

## 커리어 여정

```
C/C++ 시스템 프로그래밍 (CTI, IVR, 녹취)
        ↓
Java/Spring 모놀리식 웹 개발
        ↓
Spring Boot + REST API
        ↓
AWS 클라우드 인프라 + Docker
        ↓
DDD + MSA 패턴 학습
        ↓
AI 시대 — 직접 부딪혀보기로 결심
```

2025년 1월부터 GPT, Gemini, Perplexity, Claude를 개인 사비로 전부 결제해서 써봤습니다. 직접 비교하면서 각각의 강점과 한계를 체감했고, 최종적으로 **Claude(주) + Perplexity, Gemini(부)** 조합으로 정착했습니다.

최근에는 Claude Code Agent Team 세팅과 Skills 설정을 통해 AI 활용도를 높이는 작업을 했고, 현재는 컨텍스트 엔지니어링을 공부하면서 AI를 도구로 더 잘 쓰기 위한 스킬을 쌓고 있습니다.

매번 **"왜?"** 라는 질문이 다음 단계로 이끌었고,  **"어떻게?"** 라는 질문으로 학습을 이어갔습니다.

## 목표

C++ → Java/Spring Boot → Python → 그 다음 언어까지,
하나의 언어에 갇히지 않는 **폴리글랏 개발자**가 되는 것.

그리고 코드만 짜는 개발자가 아닌,
**구조를 설계하고 전체를 그릴 수 있는 풀스택 개발자**가 되는 것.

## 일하는 방식

같이 일하는 사람에게 가장 중요하게 보는 것은 **배우려는 의지**입니다. 겸손하게 질문하고, 끈기 있게 파고들고, 스스로 찾아서 배우는 사람과 함께 일하고 싶습니다.

저 역시 그렇게 일합니다:
- 확신이 없으면 **검증**합니다. AI가 생성한 답이든, 공식 문서의 내용이든 직접 확인합니다.
- 설계를 할 때는 **"이게 실제로 동작하는가?"** 를 항상 먼저 묻습니다.
- AI는 도구이고, 최종 판단은 사람이 합니다.

## 개인적인 이야기

저와 같이 일하는 친구가 이야기했습니다. **"스트리트 출신"** 이라고.

중소기업을 다니면서 인프라부터 이것저것 많은 경험을 했고, 지금 와서 생각하면 그 시간이 헛되지 않았던 것 같습니다. 대기업의 체계적인 교육이나 잘 나뉜 역할 분담 없이, 부딪히면서 배운 것들이 지금의 저를 만들었습니다. 그래서 "스트리트 출신"이라는 이야기가 참 좋았습니다.

저는 생각합니다. 인생을 바꾸는 건 조그만한 계기와 건강한 마인드셋, 그리고 삶을 대하는 태도와 고통을 참는 인내라고. 그래서 끊임없이 노력하는 삶을 살려고 합니다.
